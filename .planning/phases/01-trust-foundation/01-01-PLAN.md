`raw_everydb2` は物理的には `public.n_*`（確定テーブル、01-RESEARCH.md Sources で実測: スキーマは public のみ・n_* が system of record）への **VIEW** として作成（UPDATE/DELETE 不可）。主要5系統（01-CONTEXT.md D-02）の VIEW 文字列を含める: `CREATE OR REPLACE VIEW raw_everydb2.n_race AS SELECT * FROM public.n_race;` および `n_uma_race`, `n_harai`, `n_hyosu`, `n_odds_tanpuku`, `n_odds_fukusho`（存在確認は scripts/run_apply_schema.py 側で実行時 query）。

D-06 二重保護のため、`KEIBA_DB_USER`（= ETL が使う raw 読取ロール）に対して以下の SQL を**必ず発行**する:
- `REVOKE UPDATE, DELETE, TRUNCATE ON ALL TABLES IN SCHEMA raw_everydb2 FROM <KEIBA_DB_USER>;`
- `ALTER DEFAULT PRIVILEGES IN SCHEMA raw_everydb2 REVOKE UPDATE, DELETE, TRUNCATE FROM <KEIBA_DB_USER>;`

これらは plan 01-03 Task 3 の `test_raw_role_has_no_update_grant` が検証する対象と完全一致（スキーマ=`raw_everydb2`、ロール名=`KEIBA_DB_USER`）。REVOKE 未実行の場合 plan 01-03 Wave 3 で確実に fail するため、本 Task で必須として実行する。normalized/label/prediction/backtest は空 SCHEMA のみ作成（テーブル定義は後続 plan で normalized のみ作成、他は後続 Phase）。

`scripts/apply_schema.sql` に idempotent な SQL（`CREATE SCHEMA IF NOT EXISTS`, `CREATE OR REPLACE VIEW`, `REVOKE`）をまとめる。`scripts/run_apply_schema.py` を `argparse` で `--sql-file`（デフォルト `scripts/apply_schema.sql`）・`--dry-run` をサポートするエントリポイントとして作成。**管理者接続の取り扱い（D-06 必須化）:** REVOKE は所有者/superuser 権限が必要なため、`scripts/run_apply_schema.py` は `KEIBA_ADMIN_DSN` 環境変数から接続する。`KEIBA_ADMIN_DSN` 未設定時は **skip + WARNING ではなく即座に `RuntimeError("KEIBA_ADMIN_DSN is required to apply REVOKE; D-06 raw read-only protection is mandatory")` で fail する**（成功基準#2 の片肺欠落を防止。ユーザーが .env に KEIBA_ADMIN_DSN を設定するまで Task 3 は完了しない）。正常時は CREATE SCHEMA → CREATE VIEW → REVOKE の順に SQL を順次実行し、各ステップの結果を INFO ログ出力する。
