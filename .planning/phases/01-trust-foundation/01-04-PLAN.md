<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 訓練窓 → val/test データ | category map が訓練窓で fit され val/test に適用される境界。test 構成の leakage を防ぐ（§14.3） |
| train slice → calibration slice | CalibratedClassifierCV の cv='prefit' が両者を分離する境界。時系列順序が逆転すると calibration リーク（§15.2） |
| feature history → observation cutoff | merge_asof backward が cutoff 以前の履歴のみを付与する境界。未来情報が cutoff を跨ぐと feature リーク（§13） |
| train race_id → test race_id | GroupTimeSeriesSplit が race_id を分割する境界。同一 race_id の train/test またぎは §8.4 違反 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | merge_asof への未ソート入力で黙って間違った結合を返す（Pitfall 5） | mitigate | `pit_join_backward` が `obs[on_cutoff].is_monotonic_increasing` と `hist[on_asof].is_monotonic_increasing` を事前チェックし違反時 `raise ValueError`（D-17、ASVS V5/V7） |
| T-04-02 | Tampering | sklearn TimeSeriesSplit で同一 race_id が train/test にまたぐ（§8.4 違反、sklearn #19072 open） | mitigate | `race_id_time_series_split` で race_id 単位分割し `assert set(train_rids).isdisjoint(set(test_rids))` を各 fold で実行。mlxtend `GroupTimeSeriesSplit` を副 API で提供（D-17） |
| T-04-03 | Tampering | category map が test 構成で再 fit されて cardinality leakage | mitigate | `fit_category_map` は訓練窓 series のみを受け取る API 設計。`apply_category_map` は既存 code dict を受け取り再 fit しない。未知は `__UNSEEN__`・欠損は `__MISSING__` に機械フォールバック（§14.3/14.5） |
| T-04-04 | Tampering | NaN → コード -1 で LightGBM が missing と unknown を混同（§14.5 欠損理由区別違反） | mitigate | `apply_category_map` が NaN を `__MISSING__` に置換 → 別コード付与 → `.astype("int32")` で非負を保証（§14.3・test_non_negative_int32 で検証） |
| T-04-05 | Tampering | CalibratedClassifierCV(cv=5) の KFold shuffle で look-ahead leak（§15.2） | mitigate | `fit_prefit_calibrator` が `cv="prefit"` 固定 + `assert train_max_date < race_dates_calib.min()` で時系列分離を強制（D-17・test_calib_before_train_raises で検証） |
</threat_model>
