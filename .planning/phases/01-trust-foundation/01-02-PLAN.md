<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| readonly_cur → everydb2.public.n_* | 品質ゲート SELECT の境界。読取専用ロールで UPDATE/DELETE 不可（plan 01 の REVOKE） |
| reports/ → CI / 人間 | 出力レポートの境界。JSON に機密情報（DB パスワード等）を含めない（anti-pattern #20、ASVS V8） |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | silent fallback で品質チェック結果を隠蔽 | mitigate | verdict 計算を `all(r.passed for r in results if r.severity == "block")` で機械的に決定。CheckResult は dataclass でフィールド明示。未知テーブル/不明カラムは例外で fail に傾く（D-01、ASVS V7） |
| T-02-02 | Information Disclosure | reports/quality_report.json に DB 認証情報混入 | mitigate | レポート出力対象を CheckResult の name/passed/severity/detail に限定。DSN や password は一切含めない。`scripts/run_quality_report.py` で dump 前に allowlist filter を通す（ASVS V8、anti-pattern #20） |
| T-02-03 | Repudiation | 品質ゲート未実行で下流 ETL が動く | accept | Build-Order DAG で品質ゲート → ETL の順序を運用で担保。Phase 8 adversarial audit で要再検証。Phase 1 では CI の `--fail-on-block` exit 1 で防止 |
</threat_model>
