## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| read-only role → etl role | backfill は read_pool（SELECT only）と etl_pool（label schema DDL/DML）を使い分け・権限昇格なし |
| etl role → label.fukusho_label | etl role は label schema のみ UPDATE/CREATE・raw_everydb2 / public.n_* へは REVOKE で物理拒否（Phase 1 実証済） |
| staging → production swap | `DROP+RENAME` の atomic window 中に並行読者が intermediate state を見るリスク（advisory lock で直列化） |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering (raw mutation) | backfill が誤って `UPDATE raw_everydb2.n_*` を発行 | mitigate | etl role は raw へ REVOKE・`compute_raw_fingerprint(before)` / `assert_raw_unchanged(after)`・test_backfill_does_not_mutate_raw inspect 検査 |
| T-03-06 | Denial of Service | backfill 実行中に並行 reader が intermediate state を見る | mitigate | `pg_advisory_xact_lock(hashtext('label.fukusho_label'))` で transaction-scoped 直列化（CR-04(b)） |
| T-03-07 | Tampering (privilege) | reader role の SELECT GRANT が RENAME 後に欠落 | mitigate | `GRANT SELECT ON label.fukusho_label TO {reader_role}` を `Identifier` で明示再発行（HIGH #3・TO PUBLIC 不使用） |
| T-03-08 | Tampering (silent data loss) | JOIN 結合漏れで race_date が NULL のまま残る | mitigate | `SELECT count(*) ... WHERE race_date IS NOT NULL` 検証・戻り値 non_null_race_date_count==554267 で全行非 NULL 証明・Task 2 checkpoint で人手確認 |
| T-03-09 | Tampering (reproducibility) | 2回目実行で rowcount/checksum が変わる | mitigate | run_label_race_date_backfill.py が `result1 == result2` を assert（HIGH #3 idempotent verify・Phase 2 run_label_etl.py パターン） |
