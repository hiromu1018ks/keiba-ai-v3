**BLOCKER-1 解決経路（Open Question #1/#2 RESOLVED・planner が src/ev/odds_snapshot.py と src/ev/ev_rank.py と src/db/backtest_load.py を読んで確定）:**

`prediction.fukusho_prediction` テーブルは `p_fukusho_hit` + provenance（model_type/model_version/feature_snapshot_id/as_of_datetime/calib_method + PK RACE_KEY 7 + race_date/split + is_primary）のみ（src/db/schema.py L59-94）。odds/EV/rank/odds_snapshot_at 列は**すべて不存在**。一方 `backtest.fukusho_backtest`（src/db/schema.py L144-202・BACKTEST_COLUMNS L77-116）には `odds_snapshot_at`/`odds_snapshot_policy`/`odds_source_type`/`odds_missing_reason` + `EV_lower`/`EV_upper`/`recommend_rank` は存在するが、**`fuku_odds_lower`/`fuku_odds_upper`（odds 値そのもの）は存在しない**（orchestrator 検証済）。よって前回「backtest JOIN で odds 値取得」経路は構造的に不可能。

**正しい odds/EV/rank 取得経路（compute_ev_and_rank 再計算経路・RESEARCH Open Question #2 推奨・最も D-07/§19.1 に整合）:**

1. **prediction SELECT（readonly pool・is_primary=true 絞り）**: `p_fukusho_hit` + provenance（model_version/feature_snapshot_id/as_of_datetime/race_date）+ PK RACE_KEY 7（year/jyocd/kaiji/nichiji/racenum/umaban/kettonum）を取得。
2. **JODDS snapshot 取得（src/ev/odds_snapshot.py）**: `fetch_jodds(readonly_cur, years=...)` で `public.n_jodds_tanpuku` + `n_jodds_tanpukuwaku_head` JOIN（datakubun='1' 中間固定・D-01）の snapshot を取得し・`select_odds_snapshot(jodds_df, race_times_df, policy)` で `merge_asof(direction='backward', by=['race_key','umaban'])` により各馬単位の cutoff 以下最大 snapshot を選択。戻り値に `fuku_odds_lower`/`fuku_odds_upper`/`odds_snapshot_at`（選択 snapshot の happyo_datetime Timestamp・L213/L326）/`odds_source_type`/`odds_missing_reason` が含まれる（snake_case・cross-plan contract）。**未来リーク構造的不可（D-02）・HIGH-1 per-horse odds 保証**。
3. **EV/rank 再計算（src/ev/ev_rank.py::compute_ev_and_rank 純粋関数）**: prediction（p_fukusho_hit）+ JODDS snapshot（fuku_odds_lower/upper）を `(race_key, umaban)` で結合した DataFrame を `compute_ev_and_rank` に渡し `EV_lower`/`EV_upper`（§11.1 直線積 p*odds）・`recommend_rank`（§11.5 階層判定 S/A/B/C/D）を算出。
4. **odds_snapshot_policy**: CLI 引数 `--odds-snapshot-policy`（default='30min_before'・enum 30min_before/10min_before）/ UI は固定 default='30min_before'。事前固定・事後選択禁止（§11.2・BACK-04）。
5. **prediction_created_at**: prediction テーブルの `as_of_datetime`（L65・予測生成時点タイムスタンプ）を代用。
6. 馬名（horse_name）/枠番（wakuban）は `normalized.n_uma_race` から JOIN・horse_id は `kettonum` 代用（PATTERNS Data Provenance表）。競馬場名は Plan 01 の `load_jyocd_map()` で code_tables.yaml から取得。

この経路は odds を EV 判定にのみ使用しモデル特徴量には混入しない（D-07 odds 非依存確率 p_fukusho_hit + EV 分離）点で Core Value に最も整合する。

read-only 保証は Plan 01 の `tests/ui/test_readonly_guarantee.py` が src/ui/ 配下（loaders.py 含む）を検証・CLI は別途 `test_loaders_readonly.py` で parameterized query・is_primary 絞り・dsn_masked 出力を検証する。
