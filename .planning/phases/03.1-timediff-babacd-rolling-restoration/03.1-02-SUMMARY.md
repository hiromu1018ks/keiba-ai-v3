# Phase 03.1 Plan 02: persist→manifest 順序化 + exists assert + repo-root 相対パス（CR-01新） Summary

`scripts/run_feature_build.py` の manifest→persist 順序依存（CR-01新・advisory hardening）を解消し、persist→exists assert→manifest 順序化 + repo-root 相対パス正規化を行った。persist 失敗時に SHA256 一致の「完成済」manifest だけが残り category map artifact が欠損する再現性破壊状態（partial-failure）を防止する。同時に test_snapshot_repro.py の stale `rolling_timediff_mean_5` 参照を整理し、CR-01新 の順序依存回帰テストを追加した。

## What Was Built

### Task 1: persist→manifest 順序化 + exists assert + repo-root 相対パス（CR-01新）
- `scripts/run_feature_build.py` L201-228 の Step 5/6 を順序変更:
  - (旧) Step 5 write_manifest → Step 6 persist_category_maps
  - (新) Step 5 persist_category_maps → exists assert → Step 6 write_manifest
- `persist_category_maps(frozen_maps, category_map_path)` を `write_manifest` 呼出の前に移動
- persist 直後に `assert category_map_path.exists(), "persist_category_maps が category map artifact を書けなかった (CR-01新・再現性破壊防止・partial-failure 抑止)"` を新規追加
- `category_map_artifact`(L204・`f"snapshots/category_map_{args.snapshot_id}.json"`) が既に repo-root 相対であることをコメントで明記（CWD 依存性除去の副次要件は確認のみ・既に満たしていた）
- docstring の Step 5/6/7 記述を新順序に更新（persist→manifest・Pitfall 5 説明追記）
- persist が例外を投げた場合、manifest 書出処理に到達しない構造（例外伝播）で partial-failure 状態を防止
- 行番号: persist(L214) < assert exists(L218) < write_manifest(L225)

### Task 2: test_snapshot_repro.py の stale 参照整理 + CR-01新 回帰テスト追加
