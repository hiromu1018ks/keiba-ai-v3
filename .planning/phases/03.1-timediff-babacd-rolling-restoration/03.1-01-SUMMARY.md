# Phase 03.1 Plan 01: timediff/babacd カラムを normalized ETL へ raw varchar pass-through で取り込み Summary

normalized ETL に `n_uma_race.timediff(varchar4)` と `n_race` の `sibababacd`/`dirtbabacd`/`trackcd`(全varchar) を SELECT/DDL/INSERT の3定義で整合して追加し、staging-swap を DDL 駆動化して新カラムが即座に本番スキーマへ反映されるよう修正した。

## What Was Built

### Task 1: n_uma_race に timediff(varchar4) を追加
- `_UMA_RACE_SELECT_COLUMNS` / `_UMA_RACE_COLUMNS`(DDL) / `_NORMALIZED_COLUMNS["n_uma_race"]` の3定義へ `timediff` を同順序（いずれも末尾・index 43）で追加
- `_transform_uma_race_df` の `real_cols`/`int_cols` には**追加しない**（raw varchar pass-through・Pitfall 1 sentinel 0000/9999 処理は後続 Plan 03 builder 側）
- DDL は `"timediff varchar(4)"`（raw `public.n_uma_race.timediff` と同一型）

### Task 2: n_race に baba3 を追加
- `_RACE_SELECT_COLUMNS` / `_RACE_COLUMNS`(DDL) / `_NORMALIZED_COLUMNS["n_race"]` へ `syubetucd` の直後に `sibababacd`/`dirtbabacd`/`trackcd` を追加
- DDL: `sibababacd varchar(1)` / `dirtbabacd varchar(1)` / `trackcd varchar(2)`（全 varchar・cast 対象外）
- `_transform_race_df` は `kyori` 個別キャストのみで real/int cols 構造を持たないため baba3 は自動的に varchar pass-through

### Rule 3 修正: staging-swap を DDL 駆動化
- 背景: timediff/baba3 追加後、`_idempotent_load` の `CREATE TABLE ... (LIKE normalized.<table> INCLUDING ALL)` が既存 DB テーブルの**古いスキーマ**（baba3 無し）を staging にコピーし、`INSERT INTO staging (..., sibababacd, ...)` が `column "sibababacd" of relation "n_race_staging" does not exist` で失敗した
- 修正: `_TABLE_DDL_COLUMNS` mapping（`n_race`→`_RACE_COLUMNS`, `n_uma_race`→`_UMA_RACE_COLUMNS`）を導入し、staging を最新 DDL で直接生成
- `_create_table_ddl` が `*_staging` suffix を除去して PK 定義を lookup するよう対応
- `DROP TABLE IF EXISTS staging` を CREATE 前に発行（残骸衝突回避・同一トランザクション内で安全）
- raw 不変性・advisory lock・rowcount verify は不変
