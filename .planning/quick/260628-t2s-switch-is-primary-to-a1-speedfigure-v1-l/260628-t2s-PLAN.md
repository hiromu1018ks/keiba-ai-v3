## 2. 設計判断（4点・review-gate で確認）

### 判断1: A1 予測の test 窓スコープ → **Phase 4 標準（test 2024下期）**
- run_train_predict.py 標準 chain（split_3way デフォルト: train 2016-07..2023-12 / calib 2024-01..06 / **test 2024-07..12**）。
- postreview-v2（現 is_primary）と同一 test 窓スコープ構造。label は v1.1.0（universe 倍増）。
- A1 の黒字性は Spike 001 で BT窓（test 2023/24/25）証明済み。Phase 4 標準 test での保存は「デプロイ主モデル予測」の正しい姿。

### 判断2: as_of_datetime → **FIXED_REPRODUCE_TS（2026-06-20 00:00 UTC）**
- byte-reproducible 聖域。run_ablation chain と同一。postreview-v2 の非再現 as_of にはしない。

### 判断3: 既存 postreview-v2 lightgbm（is_primary=True）の取扱 → **False に UPDATE**
- set_primary_model は `feature_snapshot_id + as_of_datetime` スコープ限定で reset→set。**A1 新スコープで呼んでも既存 postreview-v2（別スコープ）の True は残る**。
- 受入基準1「他 model_version は False」を厳密に満たすため、set_primary_model 呼出とは別に **既存 postreview-v2 lightgbm を is_primary=False に UPDATE** する（D-10 人間承認対象）。lightgbm_rr は既に False で触らない。
- 結果: 全 model_version で A1 のみ is_primary=True。

### 判断4: run_train_predict.py vs 小スクリプト → **小スクリプト（run_ablation chain 流用）**
- run_train_predict.py は `_assert_snapshot_id_matches_data_module`（L174-193）で `data.py.SNAPSHOT_PATH`（postreview-v2 固定）と一致しないと `sys.exit(1)`。**A1 を扱えない**。
- また `_prepare_feature_df` が `load_feature_matrix()`（引数なし=SNAPSHOT_PATH デフォルト）を呼ぶため、args.snapshot_id を A1 にしても postreview-v2 を読んでしまう。
- → run_ablation.py と同一 chain（`load_feature_matrix(snapshot_id)` 明示 + `train_and_predict` + `load_predictions`）の **再現性ある小スクリプト**（scripts/save_primary_prediction.py・§19.1 再現性は スクリプト+snapshot+uv.lock で担保）を新設。生産スクリプト（run_train_predict.py / orchestrator）は一切変更しない。

## 3. 手順

### 手順1: A1 予測生成 + prediction 保存 + idempotent 検証（live-DB 書込）

新設 `scripts/save_primary_prediction.py`（run_ablation chain 流用・最小）:
1. `load_feature_matrix("20260625-1a-speedfigure-v1")` + `load_labels` + `build_training_frame`（label v1.1.0）
2. `train_and_predict(model_type="lightgbm", feature_snapshot_id=A1, snapshot_id=A1, version_n=1, seed=42, as_of_datetime=FIXED_REPRODUCE_TS)`（**split_periods 渡さない = Phase 4 標準**・category_map も run_train_predict 準拠で渡さない）
3. etl pool で `load_predictions` ×2 → checksum1 == checksum2 検証（idempotent）
4. model_version = `make_model_version("20260625-1a-speedfigure-v1","lightgbm",1)` = `20260625-1a-speedfigure-v1-lgb-v1`（コマンド指定と一致確認）
5. is_primary=False で保存（predict_p_fukusho 既定・D-09）

**検証**: 保存後 SELECT で A1 行が is_primary=False で存在・checksum 一致。

### 手順2: is_primary 切替（★D-10 人間承認 checkpoint・live-DB 書込）

**実行前に AskUserQuestion でユーザー承認を得る（D-10 聖域）。** 承認後、同一トランザクションで:
1. `set_primary_model(primary_model_type="lightgbm", primary_model_version="20260625-1a-speedfigure-v1-lgb-v1", feature_snapshot_id="20260625-1a-speedfigure-v1", as_of_datetime=FIXED_REPRODUCE_TS)` → A1 スコープ reset→True（post-condition: 当該スコープで1 model_type のみ True・REVIEW HIGH#7）
2. 既存 postreview-v2 lightgbm を `UPDATE prediction.fukusho_prediction SET is_primary=false WHERE model_version='20260620-1a-postreview-v2-lgb-v1'`（受入基準1・判断3）

**検証**: `SELECT model_version, is_primary FROM prediction.fukusho_prediction` で **A1 のみ is_primary=True**（postreview-v2 は False・lightgbm_rr は False 維持）。

### 手順3b: Phase 4 標準窓（test 2024下期）回収率比較（★ユーザー指示追加）
