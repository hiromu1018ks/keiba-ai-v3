## 1. 実データ検証による findings 再分類（最重要）

### CR-01 【訂正・withdrawn】'0' は「良」でなく「未設定/初期値」
`03.1-REVIEW.md` CR-01 は「`'0'`=良馬場・有効値・除外は data loss」と判断したが、**JRA-VAN JV-Data 公式仕様で `'0'`=未設定/初期値**（有効値は `'1'`=良, `'2'`=稍重, `'3'`=重, `'4'`=不良）。
- 根拠: JRA-VAN 公式フォーラム https://developer.jra-van.jp/t/topic/58 「馬場状態コードのいずれかが0になる仕様は定めていない（0は未設定）」・JV-Data コード表
- 実DB裏付け: ダート(trackcd 23-25)レースの `sibababacd` は `'0'`(未設定)が99%・芝(trackcd 10-22)レースの `sibababacd` は `'1'`(良)が最多
- 結論: `'0'` 除外（旧実装 intent）は**正しかった**。CR-01 の `'0'` 除去指摘は誤りとして **withdrawn**。

### 新Critical 【review 未検出・実DB発見】babacd trackcd 芝/ダート分岐逆転
旧実装 `turf_mask = lead.isin(["1","2"])`, `dirt_mask = lead == "5"`（先頭桁判定）は **JRA-VAN TrackCD 体系と逆転**:
- trackcd `'2x'`（実態: 23/24 = 平地ダート・最多 19,163件）を**芝扱い** → `sibababacd '0'`(未設定) → `__MISSING__`
- trackcd `'5x'`（実態: 51-59 = 障害芝・1,444件）を**ダート扱い** → `dirtbabacd` → 誤対応/`__MISSING__`
- 正: JRA-VAN トラックコード体系（10-22 平地芝 / 23-25 平地ダート / 51-59 障害=芝・JV-Data コード表 `JV-Data490.xlsx`）
- これが babacd feature data loss の**真因**（review は '0' 扱いに目を取られ分岐ロジックを見落とした）。

### WR-03 【訂正・wontfix】timediff 異常フォーマットは存在しない
実DB全件 4文字・先頭 `[+-]` で統一（`+`:509,756 / `-`:39,629 / sentinel `0000`:376 / `9999`:4,506）。先頭数字は sentinel `0000`/`9999` のみで `"1000"` 等の異常フォーマットは**不在**。実害なし → 防御的 validator は Info/任意・**wontfix**。
